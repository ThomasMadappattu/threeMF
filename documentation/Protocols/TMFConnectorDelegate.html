TMFConnectorDelegate Protocol Reference

Conforms to NSObject
Declared in TMFConnectorDelegate.h

Overview

An instance of TMFConnector uses methods in this protocol to inform the controller about discovery, subscription state changes and errors.

Tasks

Discovery

  • – threeMF:didChangeDiscoveringPeer:forChangeType:

    Gets called if a peer is found, changed or removed after starting discovery with specific capability requirements.
  • – threeMF:didChangePeer:forChangeType:

    Gets called if any peer is found, changed or removed. All peers with the same bonjour service name are reported with this method, no matter what capabilities they have.

Subscriber

  • – threeMF:didAddSubscriber:toCommand:

    Gets called if a peer subscribed to a published command
  • – threeMF:didRemoveSubscriber:fromCommand:

    Gets called if a peer got ubsusbcribed from a published command
